The Loboflix
Design System

A complete foundation for a premium, cinema-grade film planning app. Loboflix answers one question — “which film should I watch tonight?” — and this system makes every screen feel calm, intentional, and unmistakably ours.

Dark-first. Poster-led. Built on 8-point rhythm, semantic tokens, and a restrained gold accent. Everything below is production-ready guidance for design and engineering.

01 · FOUNDATIONS

Design Principles

Eight beliefs that resolve every design decision. When two options compete, the one that better serves these principles wins.

01

Content is the interface

Posters and backdrops are the heroes. Chrome recedes — no heavy frames, no competing color. The UI exists only to present the film and get out of the way.

02

Calm over dense

Few elements per screen. Generous negative space. We resist the urge to fill every pixel — breathing room is a feature, not wasted space.

03

One clear next step

Every screen has a single primary action. Hierarchy is decisive — one dominant element, everything else supporting. Never two things shouting.

04

Consistency builds trust

The same token, the same spacing, the same motion — everywhere. A user should predict how the app behaves before they touch it.

05

Rhythm through the 8-grid

All spacing, sizing, and layout snap to multiples of 8 (with a 4 half-step). Consistent rhythm reads as polish, even subconsciously.

06

Accessible by default

Every text pair clears WCAG AA. Touch targets are ≥44px. Dynamic Type, VoiceOver, and focus states aren't add-ons — they ship with the component.

07

Motion with meaning

Animation explains change — where something came from, where it went. Never decorative, never blocking. Fast, soft, and honest to physics.

08

Always respond

Every tap, load, and error gets immediate visual feedback. Skeletons over spinners, optimistic states, and honest error copy keep the user oriented.

Signature interactions
Shared-element poster → detail

The tapped poster scales and translates into the detail hero over 320ms/standard while the backdrop cross-fades in behind it. The poster is never re-loaded — it flies.

Poster press

Scale to 0.96 with a subtle brightness lift on press-down (100ms), spring back on release. Confirms touch without a ripple.

Bottom sheet

Slides up from the bottom edge (320ms/decelerate) as the scrim fades to 72%. Drag-to-dismiss follows the finger 1:1 with rubber-band resistance at the top.

Scroll parallax

The detail backdrop scrolls at 0.5× speed and darkens as content rises over it, keeping focus on the reading surface.

Reduced Motion. When the OS requests reduced motion, all translate/scale transitions become instant opacity cross-fades. No parallax, no spring, no shared-element flight.

10 · COMPONENTS

Component Library

Every component ships with the same contract: a clear purpose, defined states, documented anatomy, and do / don't guidance. Built from the foundations above — no component invents its own spacing, color, or radius.

Buttons & Actions

Button · IconButton · FAB

One primary action per view. The gold Primary button is reserved for the single most important step — usually “Add to schedule” or “Play trailer”. Everything else steps down in weight.

Primary · Secondary · Tertiary · Danger · Disabled
Sizes

Medium (44px) is the default and the minimum touch height. Large is used full-width in sheets and detail CTAs.

Icon Button & FAB

Icon buttons are 44px min with a visible container in dark UI. The FAB (56px, gold) hosts the app's single global action — “Add a film”.

✓ DO
  • Keep exactly one Primary per screen.
  • Use verb-first labels: “Play trailer”, “Add film”.
  • Icon + label share a 8px gap, 24px content padding.
✕ DON'T
  • Stack two gold buttons side by side.
  • Use gold for destructive or cancel actions.
  • Shrink touch height below 44px.

11 · PATTERNS

Layouts

Every screen follows the same skeleton: a contextual top bar, a scrolling content column on the 4-column mobile grid, and the persistent tab bar. Content leads; chrome is thin.

Home

Tonight's pick hero → “continue your week” → recommended rails. One decision above the fold.

Schedule

Week strip / calendar on top, the day's planned films as schedule cards below.

Library

Search + filter chips pinned; 3-up poster grid. Density scales up on tablet/desktop.

Detail

Full-bleed backdrop → title + ratings → primary CTA → synopsis, cast, streaming. Parallax scroll.

Profile

Avatar + name → watch stats (films, hours, streak) → taste summary → entry to settings.

Settings

Grouped lists with clear section labels. Toggles for reminders, auto-schedule, appearance.

12 · PATTERNS

Navigation Model

Flat and shallow. Four top-level destinations, a single global add action, and predictable back behavior. Users should never feel lost or more than two taps from anywhere.

Bottom navigation

Home · Schedule · Library · Profile. Always visible, tab state persists independently. Re-tapping a tab scrolls to top, then to root.

Floating action

The gold FAB “Add a film” is available on Home and Library. It opens search in a bottom sheet — the fastest path to growing the list.

Back navigation

Detail and sheets push over the current tab. Back (gesture or top-bar chevron) returns to origin with the reverse shared-element transition. Never traps the user.

Deep links

loboflix://film/{id}, /schedule/{date}, /tonight. Reminders open straight to the film detail or player.

13 · PATTERNS

Visual Language

How imagery and surfaces behave. The rule underneath everything: the artwork is the color of the app. Our chrome is neutral so posters supply the vibrancy.

Posters & Backdrops

Posters always 2:3, backdrops 16:9, never cropped off-ratio. Rounded radius-md. High-res source only — no upscaling artifacts.

Gradients & Scrims

A bottom-up black scrim guarantees text legibility over any backdrop. Gradients are functional, never decorative color washes.

Glass & Blur

Reserved for floating chrome over content — tab bar, mini player, overlay controls. 8–12px blur, subtle border. Never on static surfaces.

Shadows

Soft, diffuse, near-black. Depth in dark mode leans on surface tint first, shadow second. No hard drop-shadows or colored glows (except the gold FAB).

Illustration & Empty States

Minimal line iconography in a soft container, never full illustrations. Empty states pair one glyph, one warm sentence, one action.

#0B0B0E
Negative Space

The dark canvas is a design element. Let it breathe between rails and sections — density is earned, never default.

14 · PRACTICE

Accessibility

Accessibility is a baseline, not a feature. Every component ships accessible or it doesn't ship. Cinema is for everyone.

AA 7.2:1Contrast

Text.primary on bg.base clears AAA; secondary clears AA. Gold on dark clears AA for large text and UI. All pairs are verified — never introduce an unchecked combination.

≥44pxTouch targets

Every interactive element has a minimum 44×44px hit area, even when the visual is smaller. Adjacent targets keep an 8px gap to prevent mis-taps.

Focus & keyboard

A 3px gold focus ring on every focusable element. Logical tab order, ESC closes overlays, arrow keys move within carousels and the calendar. No keyboard traps.

Screen readers

Posters carry the film title as alt text; rating chips announce “IMDb 8.4 out of 10”. Decorative gradients are aria-hidden. Live regions announce snackbars.

Dynamic Type

Type scales with the OS setting up to 200%. Layouts reflow and truncate gracefully — no clipped labels, no fixed-height text containers.

Reduced motion & color

Honors reduce-motion (cross-fades replace transforms). Status is never color-only — a shape or label always accompanies success/error hues.

15 · PRACTICE

Design Guidelines

The habits that keep Loboflix coherent as it grows. When in doubt, choose the calmer, more consistent option.

✓ DO
  • Reach for a token — never hardcode a hex, px, or radius.
  • Give one screen one primary action.
  • Let posters supply the color; keep chrome neutral.
  • Prefer bottom sheets to center dialogs on mobile.
  • Show skeletons in the exact shape of incoming content.
✕ DON'T
  • Spread gold across many elements — it stops meaning “primary”.
  • Fill every gap; negative space is intentional.
  • Introduce a new font, color, or radius outside the scale.
  • Use color alone to convey status.
  • Block the UI with a full-screen spinner when a skeleton fits.
Consistency

One pattern per problem. If a solution exists, reuse it before inventing a variant.

Scalability

Build components to flex — variable content length, RTL, dynamic type — from day one.

Performance

Lazy-load posters, cache trailers, animate only transform & opacity. 60fps is a design requirement.
